## 使用示例

### 在代码中使用图标

```typescript
import { uiIconService, UIIconType } from '@/services/uiIconService';

// 获取图标路径
const syncIconPath = uiIconService.getIconPath('sync');
// 返回: /uiicon/cat/01.webp (如果当前主题是 cat)

// 获取图标中文标签
const label = uiIconService.getIconLabel('study');
// 返回: "学习"

// 按分组获取图标
const workIcons = uiIconService.getIconsByGroup('work');
// 返回: ['study', 'meeting', 'laptop', ...]

// 获取图标所属分组
const group = uiIconService.getIconGroup('meditation');
// 返回: 'personal'
```

### 在 React 组件中使用

```tsx
import { useUIIcon } from '@/services/uiIconService';

function MyComponent() {
  const { isCustomTheme, iconPath, fallbackPath } = useUIIcon('timer');
  
  return (
    <div>
      {isCustomTheme ? (
        <img src={iconPath} alt="Timer" onError={(e) => {
          e.currentTarget.src = fallbackPath;
        }} />
      ) : (
        <TimerIcon /> // Lucide icon
      )}
    </div>
  );
}
```

## 注意事项

1. 所有图标文件必须按照编号命名 (01-80)
2. 优先使用 WebP 格式，PNG 作为降级方案
3. 每个主题必须包含完整的 80 个图标
4. 图标尺寸建议保持一致，便于在不同场景下使用
5. 新增图标类型时，需要同步更新所有主题的图标文件
